Smart Home REST API

* * *

Purpose of our project

Designing a reusable Application Production Interface (API) for Frontend Smart Home Interfaces, which interacts with the existing postgreSQL database and can be used by other students in the future.

This task will be implemented through a REST API.

What is a REST API?

REST is short for for Representational State Transfer, a special style for Web Service architecture.

Clients send requests to a server interface, using HTTP verbs, a path to the resource, and optionally a body.

The server replies to these with the requested information, and also tells how to use the API.

Methods of the API

REST represents the database CRUD (Create, Read, Update, Delete) operations through the 4 common HTTP methods:

POST: Creates an (sub-)instance for the referred resource (values in HTTP body)

GET: Selects the information stored in the referred resource (often used with URL parameters)

PUT: Updates the referred resource (values in HTTP body)

DELETE: Deletes the referred resource

Why use Node.js?

Node is a JavaScript run-time (backend) enviroment designed to build scalable network applications.

It provides a rich library of various modules which simplifies the development of applications: NPM.

For the REST API, the main module we need is Express.
It provides an easy way to handle HTTP requests asynchronously.

Requests

This is what a client can do with our API:

  • GET /api/resources/ get all entries on the table resource
  • GET /api/resources/:uuid get all entries on a specific resource table
  • GET /api/resources/:uuid?=onlyLatest=true get the last entry of the specific resource table
  • POST /api/resources/ insert on the table resource
  • POST /api/resources/:uuid insert on a specific resource table
  • PUT /api/resources/:uuid updates the row of a specific resource on the resource table
  • DELETE /api/resources/:uuid deletes a specific resource table and its correspondent row on the resource table
