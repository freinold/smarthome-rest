What is a REST API?

REST stands for Representational State Transfer, a software architectural style that defines a set of constraints to be used for creating Web services.

Clients send requests to retrieve or modify resources, and servers send responses to these requests. The requests are made using HTTP verbs, a path to the resource and optionally a body.

Why use Node.js?

Node is a JavaScript run-time enviroment designed to build scalable network applications.

It provides a rich library of various modules which simplifies the development of applications.

Purpose of our project

Designing a reusable API for the Smart Home Interface that interacts with the postgreSQL database and can help other students in the future.

Requests

This is what a client can do with our API:

  • GET /api/resources/ get all entries on the table resource
  • GET /api/resources/:uuid get all entries on a specific resource table
  • GET /api/resources/:uuid?=onlyLatest=true get the last entry of the specific resource table
  • POST /api/resources/ insert on the table resource
  • POST /api/resources/:uuid insert on a specific resource table
  • UPDATE /api/resources/:uuid updates the row of a specific resource on the resource table
  • DELETE /api/resources/:uuid deletes a specific resource table and its correspondent row on the resource table
